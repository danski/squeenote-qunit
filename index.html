  2. Automated testing == GIANT FIGHTING ROBOTS.

  3. The benefits of testing are threefold.

  4. Firstly, the very act of writing tests forces you to formalise and rationalise your design decisions ahead of time.

  5. Secondly, a set of tests form a powerful type of documentation.

  6. And finally, tests allow you to make rapid changes in full awareness of any breakages they cause.

  7. The testing cycle

    1. Write test list
    2. Write code
    3. Write tests for code
    4. Run tests
    5. GOTO 20

  8. Testing in other languages

    • Ruby: Test::Unit, RSpec, Shoulda
    • Java: JUnit
    • Actionscript: FlexUnit
    • Cocoa/Obj-C: OCUnit
    • And billions more besides. When it comes to frameworks, an old saying applies. "There's one born every minute."
